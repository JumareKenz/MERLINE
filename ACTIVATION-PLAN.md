# Executive Priorities & Specialist Activation Plan

## Tier 1 — Immediate Activation (Phase 0)

These executives must be activated first to establish the Complete Execution Brain foundation.

| # | Executive | Domain | First Deliverable |
|---|-----------|--------|-------------------|
| 01 | Chief Product Architect (CPA) | Product | Product Vision, Feature Catalogue, User Personas |
| 02 | Principal Software Architect (PSA) | Architecture | System Architecture, Technology Stack ADRs |
| 09 | MERL Domain Expert | Methodology | Domain Model, MERL Workflows, Business Rules |
| 03 | Database Architect | Data | Logical Data Model, Entity Definitions |

### Why These Four First
- **CPA** defines WHAT we build — everything flows from product
- **PSA** defines HOW we build — architecture constrains all decisions
- **MERL Domain Expert** ensures scientific validity — without this, product is uninformed
- **Database Architect** designs the foundation — all services depend on data model

## Activation Workflow

For each activation:
1. Load the executive's system prompt
2. Define the specific scope of work with clear boundaries
3. Provide all context from decisions already made
4. Set explicit deliverables and acceptance criteria
5. Apply the CEO quality gate before accepting output
6. Record all decisions in the Complete Execution Brain
7. Update affected documentation

---

## Coordination Principles

- No executive works in isolation
- Every decision is reviewed by all affected domains
- Conflicting recommendations are escalated to CEO
- All outputs become part of the Complete Execution Brain
- Implementation never begins until planning is complete
