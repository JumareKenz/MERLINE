# Product Success Metrics Framework

## 1. North Star Metric

### Recommended: **Active Studies Generating Evidence**

**Definition**: Number of studies per month that progress from design through data collection to report generation within the platform.

**Why this metric:**
- Captures end-to-end value (not just sign-ups or logins)
- Aligns with customer success (a completed study = donor report = funding renewal)
- Drives retention (study completion requires sustained platform engagement)
- Is a leading indicator of revenue (more completed studies → more value → renew/expand)

**Targets:**
- Phase 1: 30 studies/month (across 3 pilot orgs)
- Phase 2: 500 studies/month
- Phase 3: 2,500 studies/month
- Phase 4: 10,000 studies/month
- Phase 5: 50,000 studies/month

### 7.2 Event Tracking Framework

**All product events follow a standard schema:**

```json
{
  "event": "study_completed",
  "properties": {
    "org_id": "uuid",
    "user_id": "uuid",
    "study_id": "uuid",
    "study_type": "baseline",
    "methodology": "mixed_methods",
    "submission_count": 1500,
    "enumerator_count": 25,
    "duration_days": 45,
    "$group": { "organization": "org_uuid" }
  },
  "timestamp": "2026-07-18T15:30:00Z"
}
```

**Core event categories:**
1. **Activation**: signup, org_created, study_created, form_published, first_submission
2. **Engagement**: study_viewed, form_edited, submission_made, dashboard_viewed, report_generated
3. **Adoption**: indicator_created, question_added, skip_logic_configured, sync_completed
4. **AI**: ai_suggestion_shown, ai_suggestion_accepted, ai_suggestion_rejected, ai_error
5. **Retention**: login_weekly, study_completed, user_invited, org_renewal
6. **Revenue**: subscription_started, subscription_changed, payment_received, churn_event

### 7.3 Data Governance

- PII minimized in analytics events (use UUIDs, not emails/names)
- Event data retained 24 months; aggregated data retained indefinitely
- Events tagged with environment (dev/staging/production)
- Org-level opt-out capability for telemetry
- All analytics infrastructure hosted in same region as customer data

## 8. Review Cadence & Governance

### 8.1 Metric Review Schedule

| Review | Frequency | Participants | Focus |
|--------|-----------|-------------|-------|
| Daily health check | Daily | Engineering + Ops | Uptime, error rates, sync success |
| Weekly product metrics | Weekly | Product + Engineering | Feature adoption, activation, retention trends |
| Weekly business metrics | Weekly | Leadership | Pipeline, revenue, churn |
| Monthly OKR review | Monthly | Full team | Key results progress, phase milestones |
| Quarterly strategy review | Quarterly | Exec team | North Star, phase decisions, market shifts |
| Annual planning | Annual | All stakeholders | Next phase roadmap, budget, targets |

### 8.2 Metric Ownership

| Domain | Owner |
|--------|-------|
| Product adoption metrics | Product Manager |
| Business metrics (revenue, CAC, LTV) | Product Strategy Director |
| Quality metrics (uptime, performance) | DevOps Lead |
| Data quality metrics | Data Engineering + QA |
| AI quality metrics | AI Systems Architect |
| Research impact metrics | Customer Success Director |

### 8.3 Decision Triggers

| Metric Movement | Action |
|----------------|--------|
| Activation rate drops > 10% | Investigate onboarding friction • Review sign-up flow |
| Retention drops > 5% | User interviews • Feature gap analysis • Competitive check |
| NPS drops below 30 | Customer outreach • Escalation to CEO • Root cause analysis |
| AI acceptance drops below 40% | Review prompt quality • Model performance • User training needs |
| Churn rate > 5% monthly | Win-loss analysis • Competitor threat assessment • Retention campaign |
| Sync success rate < 98% | Eng team investigation • Mobile app update (if needed) |
| Data quality score < 85% | Training push • Validation rule audit • Field team support |
