## 1. Strategic Importance

Causal inference is the core intellectual challenge of MERL. Organizations need to know not just *what* changed, but *what caused* the change. Current AI capabilities are largely correlational. This research program aims to bring causal reasoning to the platform.

**Vision**: Merline becomes the first platform that helps users design, execute, and validate causal impact evaluations using AI-assisted causal reasoning.

## 2. Causal Inference for Impact Evaluation

### 2.1 Methods Supported

| Method | Description | Data Requirements | Causal Strength | MERL Use Case |
|--------|-------------|-------------------|-----------------|---------------|
| **Randomized Controlled Trial (RCT)** | Random assignment to treatment/control | Randomization feasible | ⭐⭐⭐⭐⭐ | Program impact evaluation |
| **Difference-in-Differences (DiD)** | Compare treatment vs control over time | Pre/post data + comparison group | ⭐⭐⭐⭐ | Policy evaluation |
| **Propensity Score Matching (PSM)** | Match treated units to similar controls | Rich covariate data | ⭐⭐⭐ | Observational studies |
| **Regression Discontinuity (RD)** | Threshold-based treatment assignment | Known cutoff + density around cutoff | ⭐⭐⭐⭐ | Program eligibility thresholds |
| **Instrumental Variables (IV)** | Exogenous variation affecting treatment | Valid instrument | ⭐⭐⭐⭐ | Endogenous treatment |
| **Synthetic Control** | Weighted combination of control units | Single treated unit + multiple controls | ⭐⭐⭐⭐ | Case studies |
| **Fixed Effects** | Within-unit variation over time | Panel data | ⭐⭐⭐ | Longitudinal studies |
| **Directed Acyclic Graphs (DAGs)** | Causal diagram + conditional independence | Subject matter expertise | ⭐⭐⭐ | Confounder identification |

### 2.2 AI-Assisted Causal Inference

| Capability | Description | Technical Approach |
|------------|-------------|-------------------|
| **Method recommendation** | Suggest appropriate causal identification strategy from study context | Few-shot prompting + decision tree |
| **Confounder identification** | Identify variables that must be controlled for | LLM + causal graph reasoning |
| **DAG construction** | Build causal diagram from program theory | NLP extraction from ToC + domain knowledge |
| **Balance checking** | Assess covariate balance after matching | Automated statistical tests |
| **Robustness checks** | Recommend and execute sensitivity analyses | Rule-based + LLM recommendation |
| **Assumption documentation** | Explicitly state and test causal assumptions | Structured output from LLM reasoning |

### 2.3 Integration with Existing Workflow

```
Study Design Phase:
  Human: "I want to evaluate the impact of our training program on income"
  
  AI Causal Assistant:
    1. "Which causal method is appropriate? Based on your context (no randomization, 
       pre/post data available, comparison group exists), I recommend Difference-in-Differences."
    2. "Here are the confounders you need to measure: education, baseline income, location..."
    3. "The key assumption for DiD is parallel trends. Here's how to test it."
  
  Human: Accepts method, refines confounders
  
Analysis Phase:
  AI Causal Assistant automatically:
    1. Runs DiD specification
    2. Tests parallel trends pre-treatment
    3. Reports ATT estimate with confidence intervals
    4. Runs placebo test (randomly assign fake treatment)
    5. Reports robustness: "Result is robust to adding covariates, 
       placebo test shows no effect (p=0.42)"
```

## 3. Counterfactual Generation for Program Planning

### 3.1 Use Cases

| Use Case | Description | Value |
|----------|-------------|-------|
| **What-if simulation** | "What would outcomes look like if we allocated 20% more budget to training?" | Program optimization |
| **Counterfactual comparison** | "What would have happened without the intervention?" | Impact attribution |
| **Scenario planning** | "If drought reduces crop yield by 30%, what is the impact on food security?" | Risk management |
| **Optimal allocation** | "Where should we deploy resources for maximum impact?" | Resource optimization |

### 3.2 Technical Approach

| Method | When | Implementation |
|--------|------|----------------|
| **Structural causal models** | Known causal graph + parametric assumptions | SEM / DoWhy |
| **GAN-based counterfactuals** | High-dimensional data, complex relationships | CausalGAN / Deep-SCM |
| **LLM-generated scenarios** | Qualitative counterfactuals, planning | Structured prompting |
| **Reinforcement learning** | Sequential allocation decisions | Contextual bandits |

### 3.3 Counterfactual Validation

Counterfactuals are inherently unobservable. Validation methods:

| Method | Description | Target |
|--------|-------------|--------|
| **Backtesting** | Generate counterfactual for past period, compare to actual | Error < 10% |
| **Holdout validation** | Hide portion of treatment group, predict counterfactual | Calibration |
| **Expert review** | Domain experts evaluate plausibility | Agreement > 80% |
| **Sensitivity analysis** | Vary assumptions, measure impact on conclusions | Stability |

## 5. Neuro-Symbolic AI for MERL Domain Reasoning

### 5.1 Rationale

LLMs alone are insufficient for rigorous causal reasoning: they can hallucinate causal relationships, fail to respect structural constraints, and cannot perform exact mathematical inference. Neuro-symbolic AI combines LLM flexibility with symbolic rigor.

### 5.2 Architecture

```
┌────────────────────────────────────────────────────────────┐
│               NEURO-SYMBOLIC REASONING ENGINE                │
│                                                             │
│  Neural (LLM)                           Symbolic            │
│  ┌──────────────────────┐              ┌───────────────┐   │
│  │ • DAG proposal      │─────────────→│ • Structural  │   │
│  │ • Method selection   │              │   validation  │   │
│  │ • Confounder ID     │              │   (acyclic,   │   │
│  │ • Assumption doc     │              │    consistent)│   │
│  │ • Narrative          │              └───────┬───────┘   │
│  └──────────────────────┘                      │           │
│                                                ▼           │
│                                         ┌───────────────┐   │
│                                         │ • DoWhy       │   │
│                                         │   estimation  │   │
│                                         │ • Sensitivity │   │
│                                         │   analysis    │   │
│                                         │ • Counter-    │   │
│                                         │   factual     │   │
│                                         └───────────────┘   │
└────────────────────────────────────────────────────────────┘
```

### 5.3 Knowledge Graph Integration

MERL domain knowledge encoded as a symbolic knowledge graph:

| Entity Type | Example | Relationships |
|-------------|---------|---------------|
| Program component | "maternal health training" | CAUSES, MEDIATES, CONFOUNDS |
| Outcome | "reduced maternal mortality" | AFFECTED_BY, COVARY_WITH |
| External factor | "health system funding" | MODERATES, CONFOUNDS |
| Program context | "rural Kenya" | INTERACTS_WITH |

The knowledge graph:
1. Constrains LLM proposals (no cycles, valid cause-effect types)
2. Provides structural priors for causal learning
3. Enables reasoning over known relationships
4. Grows with each study (knowledge accumulation)

### 5.4 Example Interaction

```
User: "What is the impact of our clean water program on child diarrhea rates?"

Neuro-Symbolic Engine:
1. [LLM] Constructs initial DAG from program context:
   WaterAccess → WaterQuality → Diarrhea
   WaterAccess → HygieneBehavior → Diarrhea
   Income → WaterAccess, Diarrhea

2. [Symbolic] Validates DAG:
   ✓ Acyclic
   ✓ All edges have known structural types
   ✗ Missing confounder: "Sanitation quality"
   → LLM revises to include Sanitation

3. [Symbolic] Identifies identification strategy:
   "Based on the DAG, estimate using: 
    P(Diarrhea | do(WaterAccess)) 
    Controlling for: Income, Sanitation
    Using: IPW with propensity scores"

4. [Neural + Symbolic] Executes estimation:
   ATT = -0.12 (95% CI: -0.18 to -0.06)
   "Communities with clean water access have 12 percentage points 
    lower child diarrhea rates (p < 0.001)"
```

## 8. Open Research Questions

1. **LLM causal reasoning reliability**: Can LLMs reliably identify valid causal identification strategies? How does performance compare to human methodologists?
2. **Causal graph accuracy**: How accurate are LLM-proposed DAGs compared to expert-built DAGs in MERL contexts?
3. **Neuro-symbolic integration**: What is the optimal division of labor between neural and symbolic components?
4. **Counterfactual validation**: How can counterfactuals be validated when ground truth is unavailable?
5. **Decision outcome tracking**: What infrastructure is needed to close the predict → act → monitor loop effectively?
