# Merline Community Program

## Community Philosophy

Community transforms customers from users into advocates. A strong community reduces support costs, accelerates adoption through peer learning, generates product ideas, and creates a moat against competitors. Merline's community is built on the principle that every organization implementing MERL has valuable knowledge to share.

**Core principles:**
- Customer-to-customer learning is more trusted than vendor-to-customer
- Community contributions should be recognized and rewarded
- The community should be self-sustaining (users help users)
- Every voice matters (NGO to government, enumerator to director)

## User Community Design

### Community Platform

**Platform:** Discourse (self-hosted or cloud) — purpose-built for community forums

**Structure:**
```
Merline Community
├── 👋 Welcome (introductions, community guidelines)
├── 💬 General Discussion (open topics, MERL conversations)
├── ❓ Help & Support (user-to-user support, moderated by Merline)
├── 📚 Best Practices (shared workflows, tips, templates)
├── 📊 Show and Tell (share your studies, dashboards, reports)
├── 💡 Feature Requests (upvote and discuss product ideas)
├── 🐛 Bug Reports (report and track issues)
├── 🎓 Training & Certification (discussions, study groups)
├── 🌍 Regional Groups (East Africa, West Africa, South Asia, etc.)
├── 🔌 Integrations & API (DHIS2, Power BI, API questions)
└── 📣 Announcements (product updates, events, community news)
```

### User Roles & Permissions

| Role | Permissions | How Earned |
|------|-------------|------------|
| Visitor | Read only | Not logged in |
| Member | Read, post, reply | Account creation |
| Regular | + upvote, flag | 10+ posts with positive ratio |
| Leader | + moderate own posts, post as solution | 50+ posts, trusted by community |
| Champion | + edit wiki, mark solutions | Nominated by Merline team |
| Moderator | Full moderation | Merline staff or trusted community member |
| Admin | Full platform admin | Merline staff |

### Community Guidelines

1. **Be respectful**: Disagreement is fine, personal attacks are not
2. **Search first**: Before posting, search if your question has been answered
3. **One topic per thread**: Keeps discussions searchable and focused
4. **No self-promotion**: Share your work, not your business
5. **Provide context**: Screenshots, environment details, steps to reproduce
6. **Mark solutions**: Helps others find answers quickly
7. **No sensitive data**: Never share personal or organizational data

## User Feedback System

### Feedback Channels

| Channel | Purpose | Frequency |
|---------|---------|-----------|
| In-app feedback | Feature-specific thumbs up/down | Every interaction |
| Community Feature Requests | Product ideas with community voting | Ongoing |
| NPS survey | Overall satisfaction | Quarterly |
| CSAT survey | Support interaction satisfaction | Post-ticket |
| User interviews | Deep qualitative research | Monthly (2-3/week) |
| Product board | Structured feature prioritization | Ongoing |
| QBR feedback | Account-level strategic input | Quarterly |

### Feedback Workflow

```
Feedback Captured (any channel)
  → Categorized (Bug, Feature Request, Improvement, Praise)
  → Prioritized (by impact + frequency + strategic alignment)
  → Product Manager reviews weekly
  → Response posted (what we're doing, why, timeline)
  → User notified of status update
  → Closed-loop: "You asked, we built" communication
```

### Product Feedback Integration

- Feature requests from community feed into ProductBoard
- Top-voted features reviewed quarterly by Product team
- Roadmap items tagged with community request count
- Feature requestors notified when feature ships

## Customer Advisory Board

### Purpose

The Customer Advisory Board (CAB) provides strategic input to Merline's product direction from a diverse group of customer representatives. CAB members represent the voice of the market and help ensure Merline's roadmap aligns with real customer needs.

### CAB Structure

| Element | Description |
|---------|-------------|
| **Size** | 8-12 members |
| **Composition** | Mix of segments (NGO, government, research, consultancy), geographies, and roles |
| **Term** | 12 months (renewable once) |
| **Meeting frequency** | Quarterly (virtual), Annual (in-person) |
| **Confidentiality** | Members sign NDA for roadmap previews |

### CAB Member Profile

- Senior enough to understand organizational strategy (M&E Director, Head of Research, Program Director)
- Hands-on enough to provide detailed product feedback
- Active platform user for 6+ months
- Willing to commit 4-6 hours per quarter

### CAB Activities

| Activity | Frequency | Description |
|----------|-----------|-------------|
| Roadmap preview | Quarterly | Review and provide input on upcoming features |
| Feature prioritization exercise | Quarterly | Weighted ranking of proposed features |
| Beta testing | Ongoing | Early access to new features with structured feedback |
| Strategic discussion | Quarterly | Market trends, competitive landscape, unmet needs |
| Annual summit | Yearly | In-person strategy session + networking |

### CAB Benefits for Members

| Benefit | Description |
|---------|-------------|
| Direct influence on product roadmap | Features requested by CAB get prioritized |
| Early access to new features | Beta access before general availability |
| Executive access | Direct communication with Merline leadership |
| Peer network | Connections with other MERL leaders |
| Recognition | Listed on Merline website (with permission) |
| Annual summit expenses | Travel and accommodation covered |

## Community Launch Plan

### Phase 1: Foundation (Months 0-6)

- Launch community platform (Discourse)
- Seed with 50+ knowledge base articles
- Invite pilot customers (3 orgs) as founding members
- Establish community guidelines and moderation policies
- Hire Community Manager
- Monthly newsletter to all registered users

### Phase 2: Growth (Months 6-12)

- Launch regional user groups (East Africa, West Africa)
- Recruit Customer Advisory Board (8 members)
- Launch template library (start 10 templates)
- First community-contributed best practices published
- Implement community rewards and recognition program
- Monthly community webinars

### Phase 3: Scale (Months 12-24)

- Launch partner community portal
- First annual Merline Connect conference
- Expand to all priority regions
- Launch feature request prioritization system
- Implement AI-assisted community (smart search, suggested answers)
- Open source first SDK components

### Phase 4: Ecosystem (Months 24-36)

- Open source additional components
- Community-elected TSC members
- Partner community self-sustaining
- User-group-led events (Merline team supports, doesn't lead)
- Community NPS benchmark > 50
