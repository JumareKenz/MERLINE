# Merline Customer Onboarding Program

## Onboarding Philosophy

Onboarding transforms a new customer from "signed contract" to "confident user achieving value." It is the most critical determinant of long-term retention. Every hour invested in onboarding reduces support tickets and accelerates expansion.

**Core principles:**
- Time-to-value is the #1 onboarding metric: first submission within 2 days
- Role-based onboarding: each persona gets what they need, nothing they don't
- Progressive complexity: teach foundation first, advanced later
- Customer does: hands-on practice > passive watching
- Success milestones visible to both customer and Merline team

---

## Program Structure: 4-Week Sprint

| Week | Theme | Customer Outcome | Milestone |
|------|-------|-----------------|-----------|
| Pre-Onboarding | Welcome & Setup Readiness | Customer prepared for kickoff | Kickoff scheduled, accounts provisioned |
| Week 1 | Foundation: Admin Setup & First Project | Org configured, admin confident | Users imported, first project created |
| Week 2 | Build: Study Design & Questionnaire | First study designed and approved | Questionnaire created, reviewed, approved |
| Week 3 | Deploy: Enumerator Training & Mobile | Field team ready to collect | Mobile app installed, assignments configured |
| Week 4 | Launch: Data Collection & Go-Live | First real data flowing | First submission received, dashboard live |

---

## Pre-Onboarding (Before Week 1)

### Customer Activities
- Receive welcome email from Customer Success Manager
- Complete account setup (verify email, set password, MFA)
- Review onboarding schedule and confirm kickoff date
- Complete pre-onboarding survey: org structure, current tools, goals, digital maturity
- Identify customer project manager and executive sponsor
- Prepare user list (names, emails, roles) for import
- Ensure mobile devices are available for field testing

### Merline Activities
- Assign Customer Success Manager
- Provision organization tenant in production environment
- Send welcome package: onboarding schedule, success criteria template, pre-work checklist
- Review pre-onboarding survey responses
- Prepare implementation plan draft based on survey
- Schedule kickoff meeting with customer team

### Deliverables
- Welcome email with onboarding schedule
- Pre-onboarding survey (submitted by customer)
- User list template (completed by customer)
- Implementation plan draft (by Merline CSM)

### Success Criteria
- Customer project manager identified
- Kickoff meeting scheduled
- Pre-onboarding survey completed
- User list provided

### Duration
- 3-5 days before Week 1 start

## Week 1: Platform Overview & Admin Setup

### Day 1: Kickoff Meeting (60-90 min)

**Attendees:**
- Customer: Project Manager, Executive Sponsor, Admin/IT lead
- Merline: CSM, Implementation Lead

**Agenda:**
- Welcome and introductions
- Onboarding overview: 4-week plan, milestones, success criteria
- Platform demonstration (30-min live walkthrough of complete workflow)
- Roles and responsibilities review
- Communication channels established (Slack/Teams/Email)
- Q&A

**Outcome:** Customer team aligned on plan, expectations set, communication channels active.

### Day 2: Admin Setup Session (90 min)

**Attendees:** Customer Admin/IT lead, Customer PM, Merline CSM

**Activities:**
- Organization profile setup (name, logo, branding, timezone, language)
- Security policies (password policy, MFA enforcement, session timeout)
- User import: bulk invite from prepared list
- Role assignment (Admin, Manager, Researcher, Enumerator, Viewer)
- Team structure creation (if applicable)
- Introduction to admin dashboard and user management

**Hands-on practice:**
- Admin invites 3 test users during session
- Admin configures organization settings independently

**Outcome:** Org configured, users invited, admin comfortable with user management.

### Day 3: First Project & Study Creation (60 min)

**Attendees:** Lead Researcher/M&E Officer, Admin, Merline CSM

**Activities:**
- Create first project (with real program name and context)
- Create first study (configure type, methodology, population, timeline)
- Introduction to study lifecycle (Draft → Approved → IP → DC → Analysis → Complete)
- Overview of indicator library

**Hands-on practice:**
- Researcher creates a project and study during session
- Researcher defines 3-5 indicators

**Outcome:** First project created, study configured, indicators defined.

### Day 4: Platform Navigation & Roles Overview (60 min)

**Attendees:** All user types (researchers, supervisors, managers, enumerators — can be separate sessions)

**Activities:**
- Navigation tour (sidebar, projects, studies, settings)
- Role-specific views and permissions
- Getting Started checklist review
- Knowledge Base introduction

**Outcome:** Every user understands where to find their work in the platform.

### Day 5: Week 1 Checkpoint (30 min)

**Attendees:** Customer PM, Merline CSM

**Review:**
- Admin setup complete? (Green/Yellow/Red)
- Users invited and activating? (Count active)
- First project and study created?
- Any blockers or concerns?

**Week 1 Success Milestones:**
- [ ] Organization configured (branding, timezone, security)
- [ ] At least 50% of invited users have activated accounts
- [ ] First project and study created
- [ ] 3+ indicators defined
- [ ] Admin can independently manage users

## Week 2: Study Design & Questionnaire Creation

### Day 1: Indicator Library Deep Dive (60 min)

**Attendees:** Researchers/M&E Officers, Merline CSM

**Activities:**
- Create new indicators (name, definition, type, numerator/denominator, targets, baselines)
- Import indicators from library
- Indicator-to-question linking concept introduction
- Indicator versioning and approval workflow

**Hands-on practice:**
- Create 5-10 indicators for the study
- Set target values and baselines

**Outcome:** Complete indicator framework defined for first study.

### Day 2: Questionnaire Builder Workshop (90 min)

**Attendees:** Researchers/M&E Officers, Merline CSM

**Activities:**
- Form builder overview (question palette, canvas, properties panel)
- Question types: text, number, date, select one/multiple, Likert, GPS, photo, audio
- Creating sections and organizing questions
- Setting required fields, validation rules, and hints
- Multi-language setup

**Hands-on practice:**
- Design a 20+ question survey during session (use real questionnaire if available)
- Add skip logic to at least 3 questions
- Configure validation rules on 3+ questions

**Outcome:** Researcher can independently build a questionnaire with basic logic.

### Day 3: Advanced Questionnaire Features (60 min)

**Attendees:** Researchers/M&E Officers, Merline CSM

**Activities:**
- Skip logic / conditional branching (complex conditions)
- Repeating groups / rosters
- Calculated fields
- Indicator-to-question linking
- Form preview (mobile and web)
- Form testing / pilot mode
- Form versioning and deployment

**Hands-on practice:**
- Link 3+ questions to indicators
- Preview form in mobile layout
- Submit test submission in pilot mode

**Outcome:** Full questionnaire designed, linked to indicators, and tested.

### Day 4: Questionnaire Review & Approval (60 min)

**Attendees:** Researchers/M&E Officers, Supervisor/Manager, Merline CSM

**Activities:**
- Review workflow: draft → submit for review → approve/reject
- Peer review process demonstration
- Address review comments
- Form approval and deployment preparation

**Hands-on practice:**
- Submit questionnaire for review
- Approver reviews and approves
- Researcher deploys as active version

**Outcome:** Questionnaire approved and ready for field deployment.

### Day 5: Week 2 Checkpoint (30 min)

**Attendees:** Customer PM, Merline CSM

**Review:**
- Indicator library populated?
- Questionnaire designed, reviewed, and approved?
- Any quality or methodology concerns?

**Week 2 Success Milestones:**
- [ ] 10+ indicators defined with targets
- [ ] Questionnaire designed (30+ questions recommended)
- [ ] Skip logic and validation configured
- [ ] Questionnaire approved and versioned
- [ ] Indicators linked to questions

## Week 3: Enumerator Training & Mobile App Deployment

### Day 1: Mobile App Installation & Setup (60 min)

**Attendees:** Enumerators (can be group session + individual setup support), Supervisors, Merline CSM, Mobile Support

**Activities:**
- Download and install Merline mobile app (Google Play / APK)
- Login with test credentials
- App walkthrough: dashboard, assignments, form navigation, GPS, media capture
- Offline mode demonstration (turn off connectivity)
- Sync process: auto-sync, manual sync, sync status indicators
- Storage management and cache clearing

**Hands-on practice:**
- Each enumerator installs app and logs in
- Navigate through a test form
- Capture GPS and photo
- Submit a test submission in offline mode
- Sync test submission

**Outcome:** All enumerators have app installed and can complete a basic submission.

### Day 2: Field Workflow Training (60 min)

**Attendees:** Enumerators, Supervisors, Merline CSM

**Activities:**
- Complete survey workflow: receive assignment → navigate to household → conduct interview → capture media → submit → sync
- Partial save and resume (simulate app crash)
- Handling error states: no GPS, camera unavailable, sync failure
- Data quality expectations: what makes a "good" submission
- Battery and storage management tips

**Hands-on practice:**
- Complete 2 full test surveys (with GPS, photo)
- Simulate app crash and resume
- Handle intentional error scenarios

**Outcome:** Enumerators confident in field workflow, including offline and error states.

### Day 3: Supervisor Training (60 min)

**Attendees:** Supervisors, Merline CSM

**Activities:**
- Supervisor dashboard: team progress, enumerator stats, quality flags
- Submission review workflow: flag types, approve/reject/back-check
- Communication with enumerators (in-app messaging)
- Assignment management: create, reassign, monitor
- Data quality monitoring: duration checks, GPS consistency, straight-lining detection
- Daily progress report generation

**Hands-on practice:**
- Review test submissions as supervisor
- Approve 1, reject 1, flag 1 for back-check
- Send message to enumerator
- Create and modify assignments

**Outcome:** Supervisors can independently monitor team, review quality, and manage assignments.

### Day 4: Manager/Admin Training (60 min)

**Attendees:** Program Managers, Admin/IT, Merline CSM

**Activities:**
- Dashboard walkthrough: KPI cards, indicator tracking, RAG status, maps
- Filtering and segmentation
- Report template selection and one-click report generation
- Custom report editing (if applicable)
- Scheduled reports and automated distribution
- User management and permission review
- Audit logs and compliance reporting

**Hands-on practice:**
- Generate first report from test data
- Export as PDF and share via link
- Schedule weekly dashboard email
- Review audit log

**Outcome:** Managers can access real-time data and generate reports independently.

### Day 5: Week 3 Checkpoint + Go-Live Readiness Review (60 min)

**Attendees:** Customer PM, Executive Sponsor, All key users, Merline CSM

**Review:**
- All training completed? (Completion rates by role)
- Mobile app deployed to all enumerator devices?
- Assignment configuration ready?
- Go-live checklist status?

**Go-Live Readiness Assessment:**
- [ ] All admin/researcher training completed (100%)
- [ ] All enumerator training completed (> 90%)
- [ ] All supervisor training completed (100%)
- [ ] Mobile app installed on all enumerator devices
- [ ] Test submissions flowing through pipeline
- [ ] Dashboard showing correct data
- [ ] Report generation working
- [ ] Support channels confirmed
- [ ] Go-live date agreed

**Week 3 Success Milestones:**
- [ ] Mobile app installed on all enumerator devices
- [ ] Enumerators completed test submission (offline + sync)
- [ ] Supervisors reviewed test submissions
- [ ] Managers generated first report
- [ ] Go-live readiness confirmed

## Week 4: Data Collection Start, Dashboard Review & Go-Live

### Day 1-3: Supported Data Collection Launch

**Activities:**
- Real assignments pushed to enumerators
- First real data collected in the field
- Merline CSM provides daily check-in (Day 1: intensive, Day 2-3: lighter)
- Real-time monitoring of submissions, sync status, data quality
- Immediate issue triage and resolution
- Celebrate first 10, 50, 100 submissions

**Merline Monitoring:**
- Submission rate vs targets
- Sync success rate
- Data quality score (validation pass rate)
- Error rates (API, mobile crash, sync failure)
- Support ticket volume

### Day 4: Dashboard & Report Review Session (45 min)

**Attendees:** Program Manager, Researchers, Merline CSM

**Activities:**
- Review live dashboard with real data
- Verify indicator tracking (targets vs actuals)
- Verify RAG status logic
- Review first report generated from real data
- Discuss any gaps: missing data, incorrect mapping, display issues
- Make adjustments (filters, indicator configurations, dashboard layout)

**Outcome:** Customer confirms dashboards and reports accurately reflect their programs.

### Day 5: Go-Live Ceremony & Sign-Off (30 min)

**Attendees:** Customer PM, Executive Sponsor, Key users, Merline CSM, Merline Team Lead

**Activities:**
- Go-live milestone review
- Celebrate achievements (submissions, studies completed, etc.)
- Customer project manager signs go-live acceptance
- Transition to ongoing support and success model
- Review what to expect in the next 30-60-90 days
- Schedule first Quarterly Business Review (QBR) date

**Go-Live Sign-Off Criteria:**
- [ ] First 100+ real submissions received
- [ ] Sync success rate > 95%
- [ ] Dashboard shows accurate live data
- [ ] At least 1 report generated from real data
- [ ] No critical/open blockers
- [ ] Customer confirms readiness for independent operation

**Week 4 Success Milestones:**
- [ ] 100+ real submissions collected
- [ ] All dashboards operational with live data
- [ ] First report generated
- [ ] Go-live signed off
- [ ] Customer success plan established for next 90 days

## Post-Onboarding: 30-60-90 Day Success Plan

### Day 30 Checkpoint
- Review adoption metrics (login frequency, submissions trend, feature usage)
- First NPS/CSAT survey
- Identify power users and early champions
- Address any friction points
- Review initial value metrics (time saved, quality improvement)

### Day 60 Checkpoint
- Feature adoption deep dive (which modules used, which not)
- Training reinforcement session for low-adoption features
- Power user identification and cultivation
- Expansion discussion (additional teams, projects, modules)
- Success story assessment

### Day 90 Checkpoint / First Quarterly Business Review
- Comprehensive value review
- ROI assessment: time saved, efficiency gains, quality improvement
- Executive sponsor alignment
- Renewal readiness assessment
- Expansion planning
- Case study consideration

### Customer Team

| **Executive Sponsor** | Provides organizational authority, removes blockers, approves go-live, champions adoption | 1-2 hrs/wk (kickoff + checkpoints) |

## Escalation Path

### Tier 1: Issue Resolution during Onboarding
```
Customer user → Customer PM → Merline CSM (same day resolution)
```

### Tier 2: Technical Escalation
```
CSM cannot resolve → Support Engineer (within 4 hours)
```

### Tier 3: Critical Escalation
```
Support Engineer cannot resolve → Engineering Lead (within 2 hours)
```

### Tier 4: Executive Escalation
```
Customer Executive Sponsor → Merline CEO/CPA (business-critical, relationship risk)
```

### Escalation Triggers
| Trigger | Escalate To | Timeline |
|---------|-------------|----------|
| Onboarding milestone missed by > 3 days | CSM Manager | Same day |
| Customer satisfaction concern raised | CSM Manager | Same day |
| Technical blocker preventing go-live | Engineering Lead | 2 hours |
| Data loss or sync failure | Backend Engineering Lead | Immediate |
| Customer executive escalates | CEO / CPA | 4 hours |
| Contract or billing issue | Sales / Finance | 1 business day |

## Health Scoring During Onboarding

### Green / Yellow / Red Status

| Dimension | Green | Yellow | Red |
|-----------|-------|--------|-----|
| **Training Completion** | > 80% per role completed on schedule | 60-80% or 1-2 days behind | < 60% or > 3 days behind |
| **User Activation** | > 70% of invited users active | 40-70% active | < 40% active |
| **Feature Adoption** | Core features (study, form, collect, dashboard) in use | Some features unused | Minimal feature usage |
| **Time to Value** | First submission within 2 days | First submission within 5 days | No first submission after 7 days |
| **Support Tickets** | 0-2 tickets (informational) | 3-5 tickets (questions) | > 5 tickets or blocker issue |
| **Customer Responsiveness** | Responds within 24h, attends all sessions | Responds within 48h, misses some sessions | Unresponsive > 48h, misses key sessions |
| **Sentiment** | Positive, engaged, asking good questions | Neutral, some concerns voiced | Negative, frustrated, resistant |

### Health Score Formula
```
Health Score = (Training × 25%) + (Activation × 20%) + (TTV × 20%) + (Tickets × 15%) + (Responsiveness × 10%) + (Sentiment × 10%)

Green: >= 80
Yellow: 50-79
Red: < 50
```

### Intervention Actions
| Health Status | Action | Owner |
|---------------|--------|-------|
| **Green** | Normal cadence, celebrate wins | CSM |
| **Yellow** | Increased check-in frequency, targeted support, root cause conversation | CSM + Support |
| **Red** | Executive escalation, intensive support, possible timeline reset | CSM Manager + Customer Exec |
